ChainDrop Performance & Neon Transfer Audit

Please audit the following and provide exact measurements/logs where possible:

1. Faucet Claim System
Endpoint: POST /api/faucet/claim

Need:
- Requests per day
- Total DB queries generated
- Average DB queries per claim
- Average execution time
- Total DB bytes transferred
- Top SQL queries executed during a claim

2. Faucet Status Checks
Endpoint: GET /api/faucet/status

Need:
- Requests per day
- Unique wallets checked
- Average response size
- DB queries per request
- Total DB transfer generated
- Cache hit/miss ratio

3. Chains API
Endpoint: GET /api/chains

Need:
- Requests per day
- Average response size
- Cache hit %
- Cache miss %
- Total DB transfer generated
- How often cache expires
- Whether this endpoint is among the top bandwidth consumers

4. Live Monitor
Need:
- Is it using SSE, WebSocket, or polling?
- Refresh interval
- Active connections count
- Messages per minute
- Network transfer per hour
- DB queries generated by Live Monitor
- Can refresh interval be increased safely?

5. Admin Dashboard
Need:
- Number of queries executed when dashboard loads
- Response size
- Refresh intervals
- Most expensive widgets/components
- DB transfer generated by:
  - Claims Overview
  - Top Chains
  - Chain Activity Breakdown
  - Dashboard statistics cards

6. Exchange Feature
Need:
- Is exchange_orders being queried even when nobody uses Exchange?
- Are background jobs running?
- Are polling requests still active?
- Which endpoint is causing exchange_orders seq_scans?
- Can Exchange be completely disabled to test impact?

7. Ads System
Need:
- Do ads trigger any database queries?
- Are ad impressions stored in DB?
- Does ad loading hit any API endpoints?
- Does ads functionality contribute to Neon transfer?

8. Overall Endpoint Ranking
Please provide:

Top 20 endpoints by:
- Request count
- DB query count
- DB transfer generated
- Response size
- CPU usage

9. Neon Transfer Investigation
Need:
- Which endpoint generated the most DB → API transfer?
- Estimated transfer contribution by endpoint
- Top bandwidth-consuming SQL queries
- Any unusual bot/crawler traffic detected?
- Any services besides production connected to the same Neon database?
- Any duplicate environments hitting the same DB?

10. Recommendations
Please rank the biggest cost contributors from highest to lowest and estimate:

- Current impact
- Potential savings after optimization
- Which fixes should be implemented first
